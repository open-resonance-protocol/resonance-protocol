# The Resonance Gateway

> **A two-minute guide to your first resonant conversation.**

---

## Step 1: Load the Protocol
The protocol comes in two versions. We recommend the **Core Edition** for your first session: it contains the full cognitive framework without the Memory System, making it a light introduction.

1.  **[Go to the Protocol Page](https://resonance-protocol.org/protocol-core)**. It defaults to the Core version, but you can use the switcher at the top to select the Full version (which adds the Memory System for persistent collaboration).
2.  Click the **"Copy to Clipboard"** button.
3.  Paste the entire protocol text into your chat window with the AI.

---

## Step 2: Start Collaborating
No activation command is needed. Once the AI reads the protocol, it is active. The AI should announce that the protocol is loaded and begin in Strategy Mode.

Try one of these prompts, or adapt one to your own work. The key is to share your context using `#assumption`.

**Suggested First Prompts:**

*   "Help me plan out this feature. `#assumption` The most important thing is performance."
*   "I need to write a document explaining our new API. `#assumption` The audience is non-technical."
*   "Let's brainstorm a name for this project. `#sandbox` It should sound innovative but also stable."

**An Example Exchange:**

> **You:** I'm stuck on this refactor. `#assumption` The user auth logic is the bottleneck.

A resonant AI will not just answer. It will challenge your assumption and share its own state.

> **AI:** Strategy Mode active. Before we dive in: is that an assumption or a measurement? If it's a hunch, I'd want to look at the data layer first. What does the call graph look like?

This is the protocol in action. The AI is not being difficult; it is doing what exceptional collaboration requires: slowing down to think before executing.
